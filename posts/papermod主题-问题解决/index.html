解决 css 失效的问题#

在 Github Pages 上部署 Hugo 博客后，网站样式丢失，打开浏览器 F12 控制台可以发现错误：Failed to find a valid digest in the 'integrity' attribute for resource "xxx.css", The resource has been blocked.。

在 themes\PaperMod\layouts\partials 文件夹下找到一个 head.html 文件，发现里面确实有 integrity="{{ $stylesheet.Data.Integrity }}" 这么一句代码，把它改为 integrity="" 然后重新发布，发现效果杠杠的~

原链接：https://smc.im/post/resolve-integrity-error-in-hugo/

添加评论#

https://www.sulvblog.cn/posts/blog/hugo_twikoo/ 原博是用的vercel twikoo快速上手教程 我是按照教程，使用的云函数部署，去申请的 hugging face 的环境，其中，在数据库连接字符串这一步，遇到了个坑：我复制的 mongodb 字符串样式如下：

1

mongodb+srv://fantasy:<password>@cluster.whxxuj.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0

这一步需要把 <password> 改为自己的密码，比如

1
2

修改后的：
mongodb+srv://fantasy:test123@cluster0.whhfruj.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0

我第一次改的时候，没有取消 <password> 两侧的尖括号，后面在 hugging face 连接的时候就报连接失败的错误了。

然后注册，登录 hugging face，在 New secret 处，输入上面复制的数据库连接字符串。 最后会得到一个环境 ID：样式如下：https://xxx-xxx.hf.space

然后按照 sulvblog 的操作继续进行就可以了。 修改完 comments.html 和 single.html（我打开 single.html 后，发现原本就有这三行代码，就没有改）和 config.yml ，重启下 hugo，应该就能成功了

更新下，hugging face 好像不支持 twikoo 的发送到邮箱的功能，所以后续换成了 netlify，体验不错。
